Le problème

Ça vous parle ?

Si vous avez plusieurs serveurs à la maison, vous connaissez la galère.

😤
Jongler entre plusieurs interfaces

Film sur le serveur du bureau, série sur le Raspberry du salon, documentaires chez les parents. Aucun endroit pour tout voir d'un coup.

💾
Copier les fichiers = impensable

Dupliquer 2 To de films juste pour centraliser ? Aucun sens. Espace gâché, synchronisation impossible, double bande passante.

🤯
La famille ne comprend pas

« C'est sur quel Emby ce soir ? » Expliquer l'infra à votre famille, c'est perdu d'avance. Il faut que ça soit transparent.

↓
VirtualLib résout tout ça en un plugin.

Une seule interface Emby. Tout votre contenu. Partout. Automatiquement.

Comment ça marche

Simple comme bonjour

Six étapes, une fois. Ensuite c'est automatique pour toujours.

01
🔌
Installez le plugin

Copiez VirtualLib.dll dans le dossier plugins d'Emby A. Redémarrez. C'est tout.

5 minutes
02
🔗
Ajoutez vos serveurs

Dashboard → VirtualLib → Paramètres. URL + clé API de chaque source. Test de connexion en un clic.

Emby / Jellyfin / Plex
03
📚
Choisissez vos bibliothèques

La liste des bibliothèques disponibles s'affiche. Cochez celles à importer. Films ? Séries ? Les deux ? Votre choix.

Granularité par bibliothèque
04
⚡
VirtualLib synchronise

Génère des fichiers .strm (~100 octets chacun) et .nfo localement. Aucun média n'est copié.

Zéro duplication
05
🏠
Ajoutez dans Emby

Pointez une bibliothèque Emby vers le dossier généré. Emby indexe tout comme si c'était local.

Natif dans Emby
06
▶️
Appuyez sur Play

Le ProxyController pipe le stream. Seek natif, transcodage à la volée, clés API jamais exposées. 100% transparent.

Proxy Range-aware
Fonctionnalités

Tout y est.
Sans compromis.

VirtualLib ne fait pas de concessions sur ce qui compte vraiment.

🎭
Bibliothèques virtuelles natives

Vos médias distants apparaissent comme des bibliothèques locales. Navigation, filtres, recherche — exactement comme si les fichiers étaient chez vous.

Expérience identique
🔄
Proxy transparent + Seek natif

Le ProxyController forwarde les Range headers HTTP. Seek au milieu d'un film = instantané. Emby A peut transcoder avant d'envoyer au client.

206 Partial Content
🔒
Vos clés API restent secrètes

Les tokens d'accès aux serveurs sources ne sont jamais dans les .strm, jamais exposés au client. Le proxy valide chaque requête contre l'index local.

Aucun proxy ouvert
📡
Sync automatique & delta

Nouveau film ajouté sur le Serveur B ? Dans l'intervalle configuré (défaut 6h), il apparaît automatiquement dans Emby A. Sans rien faire.

Détection ajouts/suppressions
🗂️
Métadonnées sans re-scraping

Les .nfo embarquent titre, synopsis, rating, genres, IDs IMDb/TMDb. Emby indexe directement sans appeler de scrapers externes.

Format NFO Kodi/Emby
🌐
Multi-serveurs illimité

Ajoutez autant de serveurs que vous voulez. Chaque connecteur est indépendant avec son intervalle de sync et sa sélection de bibliothèques.

N serveurs simultanés
Compatibilité

Fonctionne avec
les trois grands

Une interface commune, trois connecteurs. Même expérience quelle que soit la source.

🖥️
Emby
✓ Supporté — Phase 1

API REST native, pagination gérée, artwork, stream vidéo. Authentification par clé API ou token MediaBrowser.

🐟
Jellyfin
⏳ Phase 3

API compatible avec Emby. Le JellyfinConnector sera un dérivé direct d'EmbyConnector avec ajustements mineurs.

🟠
Plex
⏳ Phase 3

API XML différente. PlexConnector traduira les réponses XML vers les modèles normalisés VirtualLib.

Démarrer

Prêt en 5 minutes.
Gratuit pour toujours.

Pas d'inscription. Pas de serveur cloud. Vos données restent chez vous.

01

Télécharger VirtualLib.dll

Depuis la page Releases du dépôt GitHub.

github.com/CCoupel/Emby_VirtualLib/releases
02

Copier dans le dossier plugins Emby

Puis redémarrez votre serveur Emby.

{EmbyData}/plugins/VirtualLib/VirtualLib.dll
03

Configurer depuis le Dashboard Emby

Dashboard → Plugins → VirtualLib → Paramètres → Ajouter un serveur.

04

Synchroniser & profiter

Cliquez "Synchroniser maintenant" → Ajoutez le dossier généré comme bibliothèque dans Emby.
