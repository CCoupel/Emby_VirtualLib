Fonctionnalités

Tout ce dont vous
avez besoin

Un plugin léger qui centralise vos serveurs médias sans compromis sur la performance ni la sécurité.

Bibliothèques virtuelles natives
Les médias distants apparaissent comme des bibliothèques locales dans Emby. Navigation et lecture identiques à une source locale.
Proxy transparent
Le stream transite par le serveur hôte via ProxyController. Emby A peut transcoder à la volée. Support Range headers pour le seek natif.
Multi-sources
Emby, Jellyfin et Plex supportés via une interface commune IMediaServerConnector. Ajoutez N serveurs simultanément.
Sync automatique
Détection delta des ajouts et suppressions. Intervalle configurable (défaut 6h). Déclenchable manuellement ou via API REST.
Sécurité intégrée
Les API keys des serveurs sources ne sont jamais exposées aux clients. Le proxy ne forwarde que les items indexés — pas de proxy ouvert.
Zéro duplication
Seuls des fichiers .strm (pointeurs URL) et .nfo (métadonnées XML) sont générés localement. Aucun fichier média copié.
Architecture

Flux de données

Lors de la navigation, Emby A sert les métadonnées locales. Lors de la lecture, le ProxyController pipe le stream depuis le serveur source.

Installation

Démarrer en
quelques minutes

01

Télécharger le plugin

Récupérer VirtualLib.dll depuis la page des releases GitHub.

github.com/CCoupel/Emby_VirtualLib/releases
02

Copier dans le répertoire plugins

Placer le fichier dans le dossier plugins d'Emby, puis redémarrer le serveur.

{EmbyData}/plugins/VirtualLib/VirtualLib.dll
03

Configurer un serveur source

Dashboard → Plugins → VirtualLib → Paramètres. Ajouter l'URL + API key du serveur source.

04

Sélectionner les bibliothèques

Après test de connexion, cocher les bibliothèques distantes à synchroniser.

05

Lancer la sync & ajouter la bibliothèque

Cliquer "Synchroniser maintenant", puis ajouter le dossier généré comme bibliothèque dans Emby.

/media/virtual-libraries/ServeurB_Films/
